# 暖宠星球 - 用户数据隔离机制实现

## 实现概述

为暖宠星球微信小程序添加了完整的用户认证和数据隔离机制，确保每个用户的数据独立、安全。

## 主要功能

### 1. 用户登录与 Token 机制

- ✅ `wx.login()` 获取微信登录凭证 code
- ✅ 调用后端接口用 code 换取 uid + token
- ✅ Token 存储在本地（可加密）
- ✅ `checkAuth()` 检查登录状态
- ✅ `refreshToken()` 刷新 token
- ✅ `logout()` 退出登录

### 2. 网络请求封装 (`utils/request.js`)

- ✅ 封装 `wx.request`，自动携带 `Authorization: Bearer {token}`
- ✅ 请求前自动检查 token
- ✅ 401 状态码自动刷新 token 并重试
- ✅ 统一错误处理
- ✅ Mock 数据模式支持
- ✅ 文件上传支持

### 3. 全局状态管理 (`app.js`)

```javascript
globalData: {
  uid: '',           // 用户唯一ID
  token: '',         // 认证token
  refreshToken: '',  // 刷新token
  userInfo: null,    // 用户信息
  isLoggedIn: false  // 登录状态
}
```

### 4. 页面级改造

| 页面 | 改动 |
|------|------|
| `pages/index/index` | 点赞检查登录 |
| `pages/square/square` | 点赞、发布检查登录 |
| `pages/profile/profile` | 登录状态展示、登录/退出按钮 |
| `pages/post-detail` | 点赞、评论、收藏检查登录 |
| `pages/publish` | 发布检查登录，关联 uid |

### 5. 登录页面 (`pages/login`)

- ✅ 微信一键登录
- ✅ 手机号登录（预留）
- ✅ 隐私政策勾选
- ✅ 静默登录失败时引导登录

## 使用方法

### 静默登录（首次打开）

```javascript
// 在 app.js 的 onLaunch 中自动调用
app.silentLogin().then(res => {
  console.log('登录成功', res.uid)
})
```

### 检查登录状态

```javascript
const app = getApp()

if (app.checkAuth()) {
  const uid = app.getUid()
  // 已登录
} else {
  // 未登录
}
```

### 发起带认证的请求

```javascript
const request = require('./utils/request.js')

// GET 请求
request.get('/api/user/profile')
  .then(res => console.log(res.data))

// POST 请求
request.post('/api/posts/like', { postId: '123' })
  .then(res => console.log(res.data))
```

### 登录引导

```javascript
const app = getApp()

// 需要登录时调用，会弹出登录提示
if (!app.requireLogin()) {
  return
}
```

## 数据隔离原理

```
┌─────────────┐     POST /api/auth/login     ┌─────────────┐
│   小程序    │  ────────────────────────▶   │   后端服务  │
│             │                               │             │
│  wx.login() │  ◀────────────────────────  │  生成 uid   │
│  获取 code  │     返回 uid + token         │  生成 token │
│             │                               │             │
└─────────────┘                               └─────────────┘
         │
         │ 存储 token
         ▼
┌─────────────┐
│  本地存储   │
│  user_uid   │
│  auth_token │
└─────────────┘
         │
         │ 后续请求
         ▼
┌─────────────┐
│   请求头    │
│ Authorization│
│ : Bearer xxx │
└─────────────┘
         │
         ▼ 后端解析
┌─────────────────────────────┐
│  后端以 token 解析的 uid    │
│  为准进行数据隔离            │
└─────────────────────────────┘
```

## 后端对接指南

### 1. 配置 API 地址

编辑 `miniprogram/utils/request.js`：

```javascript
const API_BASE_URL = 'https://your-api-domain.com'  // 替换为真实地址
```

### 2. 关闭 Mock 模式

```javascript
const USE_MOCK = false  // 生产环境设为 false
```

### 3. 实现后端接口

参考 `API接口文档.md` 实现以下接口：

- `POST /api/auth/login` - 登录
- `POST /api/auth/refresh` - 刷新 token
- `GET /api/user/profile` - 获取用户信息
- 其他业务接口...

### 4. 数据隔离验证

在后端实现中，确保：

```javascript
// 从 token 中解析 uid
const uid = parseToken(token).uid

// 查询用户数据时以此 uid 为准
const userPosts = db.query('posts', { uid })  // ✅ 正确
// 不要用前端传的 uid！
```

## 注意事项

1. **Mock 模式**：当前所有请求使用 Mock 数据，方便前端开发
2. **静默登录**：首次打开静默进行，失败时显示提示
3. **登录不阻断**：未登录用户仍可浏览，点赞/评论等操作时才要求登录
4. **Token 安全**：生产环境建议对本地存储的 token 进行简单加密
