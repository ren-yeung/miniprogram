# 暖宠星球 - 云开发部署指南

## 概述

本文档说明如何将暖宠星球小程序从 Mock 模式切换到微信云开发模式。

---

## 一、环境配置

### 1.1 查看云开发环境ID

1. 打开微信开发者工具
2. 点击左侧「云开发」按钮
3. 进入云开发控制台
4. 点击「设置」- 「环境设置」
5. 复制「环境ID」（格式如：`nuanchong-xxxx`）

### 1.2 替换环境ID

需要替换以下文件中的环境ID：

| 文件路径 | 占位符 | 说明 |
|---------|--------|------|
| `project.config.json` | 无需修改 | 云函数目录配置 |
| `miniprogram/app.js` | `'yangchangren-d7gzeie6f61d60403'` | 云开发初始化 |
| `cloudfunctions/login/index.js` | `'yangchangren-d7gzeie6f61d60403'` | 登录云函数 |
| `cloudfunctions/user/index.js` | `'yangchangren-d7gzeie6f61d60403'` | 用户云函数 |
| `cloudfunctions/posts/index.js` | `'yangchangren-d7gzeie6f61d60403'` | 帖子云函数 |

**替换方法：**
将所有 `'yangchangren-d7gzeie6f61d60403'` 替换为你的实际环境ID

## 二、云数据库集合创建

### 2.1 需要创建的集合

```javascript

// 权限设置：所有用户可读，仅创建者可写
{
  "_id": "自动生成",
  "_openid": "微信openid",
  "uid": "用户唯一标识，如 NC_ABCD1234",
  "name": "昵称",
  "avatar": "头像URL（云存储fileID）",
  "phone": "手机号",
  "address": "收件地址",
  "bio": "个人简介",
  "city": "城市",
  "isVip": false,
  "coins": 0,
  "balance": "0.00",
  "followers": 0,
  "following": 0,
  "likes": 0,
  "posts": 0,
  "createdAt": 1234567890123,
  "updatedAt": 1234567890123
}
```

#### posts 集合（帖子表）

```javascript
// 集合名称：posts
// 权限设置：所有用户可读，仅创建者可写

  "uid": "发布者uid",
  "title": "标题",

  "category": "分类（晒宠/知识/活动等）",
  "likes": 0,
  "comments": 0,
  "createdAt": 1234567890123,
  "updatedAt": 1234567890123

```

#### likes 集合（点赞表）

```javascript
// 集合名称：likes
// 权限设置：所有用户可读，仅创建者可写
{
  "_id": "自动生成",
  "_openid": "微信openid",
  "uid": "用户uid",
  "postId": "帖子ID",
  "createdAt": 1234567890123

```

#### comments 集合（评论表）

```javascript
// 集合名称：comments
// 权限设置：所有用户可读，仅创建者可写
{
  "_id": "自动生成",
  "_openid": "微信openid",
  "uid": "用户uid",
  "postId": "帖子ID",

  "createdAt": 1234567890123

```

### 2.2 集合权限设置

在云开发控制台中，为每个集合设置权限：

```
// 数据权限选择：
// ✅ 使用安全规则

// 安全规则：
{
  "read": true,
  "create": "doc._openid == auth.openid",
  "update": "doc._openid == auth.openid",
  "delete": "doc._openid == auth.openid"
}
```

或者使用简化的「仅创建者可读写」权限。

## 三、云函数部署

### 3.1 部署步骤

1. 在微信开发者工具中，右键点击 `cloudfunctions` 文件夹
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成

### 3.2 部署顺序

建议按以下顺序部署：

1. **login** - 登录云函数（其他云函数可能依赖）
2. **user** - 用户云函数
3. **posts** - 帖子云函数

### 3.3 验证云函数

部署完成后，可以右键点击单个云函数，选择「测试」验证功能。

## 六、常见问题

### Q1: 云函数调用失败，提示 "not found"

**原因**：云函数未部署到云端

**解决**：
1. 确保在微信开发者工具中右键 `cloudfunctions` 文件夹
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成（通常需要 1-2 分钟）

### Q2: 云函数调用成功但数据操作失败

**原因**：集合权限配置问题

**解决**：
1. 打开云开发控制台
2. 检查集合权限设置
3. 确保数据权限设置正确

### Q3: 图片上传失败

**原因**：云存储权限或容量问题

**解决**：
1. 检查云存储容量是否充足
2. 检查图片大小是否超过限制（单文件最大 5MB）
3. 确保云存储权限允许上传

### Q4: 如何查看云函数日志

**解决**：
1. 在微信开发者工具中右键云函数
2. 选择「查看日志」
3. 可以查看调用记录和错误信息

### Q5: 如何切换回 Mock 模式

**解决**：
1. 修改 `miniprogram/utils/request.js`
2. 将 `USE_MOCK` 改为 `true`
3. 将 `USE_CLOUD` 改为 `false`

## 七、开发建议

### 7.1 数据安全

- 不要在客户端存储敏感信息
- 用户密码、手机号等敏感数据应加密存储
- 合理设置集合权限

### 7.2 性能优化

- 使用分页加载数据，避免一次加载过多
- 图片使用压缩后上传
- 合理使用云函数缓存

### 7.3 日志记录

- 在关键操作处添加 console.log
- 使用云开发控制台的日志功能调试
